# Engagement Process вЂ” Design Spec
*2026-04-24*

## Overview

Build the engagement function at Fimiliar as a proper business unit вЂ” strategy, process, and per-client context all living in francoisOS. Activity tracking (connections sent, comments, DMs) stays in Excel for the MVP. Structure is intentionally lean (strangler fig) вЂ” complexity added as the operation matures and patterns emerge.

All daily execution is driven by Claude Code skills (`/engagement-log`, `/outreach`, etc.) reading from these files.

## Master Process Doc (`README.md`)

Contains:
- **Pipeline stages** вЂ” List в†’ Approval в†’ Connections в†’ Intent Monitoring в†’ Engagement в†’ DMs
- **Cadences** вЂ” connections daily, list building monthly, touch/signal thresholds (TBD)
- **Rules** вЂ” LinkedIn limits (max 200 connections/week), what qualifies as a warmed prospect, when to move to DM
- **Link to Excel tracker** вЂ” activity tracking lives there for now

This is the single reference for how the engagement function operates. Updated as thresholds and rules are established through practice.

## Per-Client File Structure

Each `client-name.md` contains:

| Section | Contents |
|---|---|
| **Overview** | Who they are вЂ” role, company, goals |
| **ICP** | Who we're targeting вЂ” seniority, industry, geography, keywords |
| **Voice & Style** | How to write as/for them вЂ” tone, phrases to use/avoid |
| **Strategy** | Current focus, what we're doing and why |
| **Target list** | Where the list lives (Sales Nav search URL or Clay table link) |
| **Next actions** | What needs doing вЂ” the working to-do for this client |
| **Notes** | Preferences, sensitivities, client history, anything contextual |

No activity log in client files вЂ” that's Excel. This file is the brief and the strategy record.

## Skills Integration

All daily execution is Claude Code skill-driven:

| Skill | Purpose |
|---|---|
| `/engagement-log` | Daily commenting workflow вЂ” reads client files, guides session per client |
| `/outreach` | DM workflow вЂ” surfaces warmed prospects, guides outreach per client |
| `/list-build` | Monthly list building per client |
| `/weekly-report` | End-of-week performance summary across all clients |

Skills read `projects/engagement/README.md` for process rules and `projects/engagement/clients/client-name.md` for per-client context.

---

## MVP Scope

**In scope:**
- Master process doc with pipeline and rules (TBD sections clearly marked)
- All 14 client files (individuals + Legion + FrontlineXP + Fimiliar internal)
- `/engagement-log` skill (daily commenting)
- `/outreach` skill (DM workflow)

**Out of scope for MVP (add as operation matures):**
- Automated stats pulled from Excel into client files
- Signal scoring / intent thresholds (defined through practice first)
- `/list-build` and `/weekly-report` skills
- fimiliarOS integration (separate repo, future)

---

## Principles

- **francoisOS is the brain, Excel is the ledger** вЂ” strategy and context here, activity data there (for now)
- **Strangler fig** вЂ” start simple, add layers as patterns emerge; don't over-engineer before you understand the operation
- **Skill-driven execution** вЂ” every recurring workflow gets a Claude Code skill; no manual process that could be systematised
